# Widget
Виджет показывает несколько последних успешных банковских операций клиента. 

## Проект № 1
Проект, который на бэкенде будет готовить данные для отображения в новом виджете.

* `mine.py`- отвечает за основную логику проекта и связывает функциональности между собой
* `masks.py` - принимает на вход номер карты или счета и возвращает ее/его маску
* `widget.py` - модуль для работы с новыми возможностями приложения
* `processing.py` - сортирует и фильтрует список операций
* `generators.py` - фильтрует список операций по валюте, возвращает описание каждой операции, генерирует номера карт
* `decorators.py` - выполняет логи для модулей, в том числе в файл
* `utils.py` - принимает файлы типов: JSON, CSV, XLSX и возвращает список словарей с данными о финансовых транзакциях, ведется лог в соответствующую папку
* `external_api.py` - принимает на вход транзакцию и возвращает сумму транзакции в рублях

## Примеры работы функций
### get_mask_card_number:
Принимает номер карты и возвращает ее маску:
* на входе - 7000792289606361
* на выходе - 7000 79** **** 6361
### card_number_generator:
Принимая начальные значения для диапазона выдает номера банковских карт в формате XXXX XXXX XXXX XXXX, может сгенерировать номера карт в заданном диапазоне от 0000 0000 0000 0001 до 9999 9999 9999 9999
### log:
Добавляет время начала и окончания работы функции, имя, результат или ошибку
### get_transaction_amount:
Принимает на вход транзакцию и возвращает сумму транзакции в рублях
### convert_apilayer_data:
Обращается к API apilayer.com для конвертации валюты
### load_json_data, load_csv_data, load_xlsx_data
Функция принимает файл и возвращает список словарей с данными о финансовых транзакциях
## Логгирование
Выполняется запись логов в папку Logs, включая метку времени, название модуля, уровень серьезности и сообщение, описывающее событие или ошибку, которые произошли для модулей:
* utils
* masks
## Тестирование

Функциональный код покрыт тестами на 100%
